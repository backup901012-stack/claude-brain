> 2026-03-30 建立 | 全部 6 個 Phase 已完成

---

## 一、問題背景

Jamie 有 4 個自動化專案（宏觀日報、股票量化、香港賽馬、命理系統），每個專案各自獨立，遇到相同問題：
- GitHub Actions 超時（股票篩選 25 分鐘逼近 30 分鐘限制）
- 數據存 Git repo 導致膨脹（每天 3-4MB JSON，一年 1GB+）
- 每個專案重複造輪子（各自一套 email、報告生成）
- 電腦不開就不能自動執行
- 無法跨專案查詢數據
- 新專案又要重新搭建一次

---

## 七、技術選型決策

| 決策 | 選擇 | 原因 |
|:---|:---|:---|
| 資料庫 | Cloudflare D1 | 已部署、免費、SQL 熟悉 |
| 檔案存儲 | Cloudflare R2 | 同生態系、S3 相容、免費 |
| API | Cloudflare Workers | 已部署、代碼已有、免費 |
| 前端 | Cloudflare Pages | 同生態系、免費、自動部署 |
| 計算 | GitHub Actions matrix | 零成本、4 市場平行 |
| 共用套件 | Python pip via Git | Jamie 熟悉 Python |
| 前端框架 | Alpine.js + Tailwind | 零編譯、學習曲線最低 |

**核心哲學：全部押注 Cloudflare 生態系** — 減少帳號管理、統一計費、服務間零延遲。
