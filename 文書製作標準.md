## 第一部分：設計哲學

### 1. Tufte 六原則（資訊可視化之父）

| 原則 | 說明 | 落地做法 |
|:---|:---|:---|
| **最大化數據墨水比** | 圖上每一滴墨水都要承載數據 | 刪除多餘裝飾線、3D 效果、背景圖案 |
| **消除圖表垃圾** | 不承載資訊的元素都是垃圾 | 不用漸層填色、不用陰影、不用 3D 柱狀圖 |
| **高資訊密度** | 在合理範圍內最大化數據量 | 用迷你圖（sparkline）、小倍數圖、多維表格 |
| **圖形誠實** | 視覺比例必須等於數據比例 | 柱狀圖 Y 軸從 0 開始、不裁剪誤導 |
| **整合文字與數據** | 文字和圖表放在一起，不分開 | 表格旁直接寫分析，不另開一頁 |
| **尊重讀者智慧** | 讀者能處理複雜資訊 | 不過度簡化，保留有價值的細節 |

### 2. McKinsey/BCG 簡報原則

| 原則 | 說明 |
|:---|:---|
| **金字塔結構** | 結論先行，再展開論據 |
| **MECE** | 互不重疊、完全窮盡 |
| **Action Title** | 每頁標題是結論句，不是描述句（「市場下跌 2%」而非「市場概覽」）|
| **一頁一訊息** | 每頁只傳達一個核心觀點 |
| **數據驅動** | 每個結論都有數據支撐 |
| **極簡設計** | 顏色 ≤ 3 種主色、字體 ≤ 2 種、無動畫 |

### 3. 投行報告設計標準

| 項目 | 標準 |
|:---|:---|
| 正文字體 | 無襯線（標題）+ 襯線或無襯線（正文），大小 9-11pt |
| 行距 | 正文 1.3-1.5 倍、表格 1.1-1.2 倍 |
| 每行字數 | 30-75 字元（中文 15-38 字） |
| 配色 | 深藍+白+灰為主，紅綠僅用於漲跌 |
| 表格 | 斑馬紋交替行、表頭深色背景、數字右對齊 |
| 留白 | 邊距 ≥ 15mm，元素間距 ≥ 4mm |

## 第二部分：中文排版（CJK 專項）

### 字體選擇

| 平台 | 推薦字體（優先順序） | 用途 |
|:---|:---|:---|
| **macOS** | PingFang TC → Arial Unicode → STHeiti | 正文+標題 |
| **Linux (GitHub Actions)** | Noto Sans CJK TC → WenQuanYi | 正文+標題 |
| **PDF (ReportLab)** | 註冊 TTF/TTC，見下方程式碼 | 全部 |

### CJK 排版規則

| 規則 | 說明 |
|:---|:---|
| **行距 170%** | CJK 字元筆畫密，需要更多呼吸空間（正文 9.5pt → leading 16pt）|
| **段落間距** | 段落之間留 0.5-1 行空白 |
| **中英混排** | 中英文之間自動加半角空格（或用 CSS `word-spacing`）|
| **標點禁則** | 句號、逗號不出現在行首；左括號不出現在行尾 |
| **Emoji 清理** | 所有文字必須經過 `remove_emoji()` 過濾 BMP 以外字元 |

### 字體註冊（統一程式碼）

```python
import os, platform
from reportlab.pdfbase import pdfmetrics
from reportlab.pdfbase.ttfonts import TTFont

def register_font():
    """跨平台中文字體註冊 — 所有專案統一使用此函數"""
    candidates = {
        "Darwin": [  # macOS
            ("/Library/Fonts/Arial Unicode.ttf", "ChineseFont"),
            ("/System/Library/Fonts/PingFang.ttc", "ChineseFont"),
            ("/System/Library/Fonts/STHeiti Medium.ttc", "ChineseFont"),
        ],
        "Linux": [  # GitHub Actions
            ("/usr/share/fonts/opentype/noto/NotoSansCJK-Regular.ttc", "ChineseFont"),
            ("/usr/share/fonts/truetype/noto/NotoSansCJK-Regular.ttc", "ChineseFont"),
            ("/usr/share/fonts/opentype/noto/NotoSansCJKtc-Regular.otf", "ChineseFont"),
        ],
    }
    system = platform.system()
    for path, name in candidates.get(system, candidates["Linux"]):
        if os.path.exists(path):
            pdfmetrics.registerFont(TTFont(name, path, subfontIndex=0))
            return name
    # 最後備援
    from reportlab.pdfbase.cidfonts import UnicodeCIDFont
    pdfmetrics.registerFont(UnicodeCIDFont('STSong-Light'))
    return 'STSong-Light'
```

### Emoji 清理（統一程式碼）

```python
import re

def remove_emoji(text):
    """清除所有可能導致 ◆◆ 亂碼的字元"""
    if not text:
        return text
    return re.compile(
        "[\U00010000-\U0010FFFF]"  # BMP 以外（emoji 等）
        "|[\u2600-\u27BF]"          # 雜項符號
        "|[\uFE00-\uFE0F]"          # 變體選擇符
        "|[\u200D\u200B\uFEFF]"     # 零寬字元
    ).sub("", text)
```

## 第三部分：PDF 報告規範

### A4 版面參數（不可違反）

```
頁面：595pt × 842pt (A4)
邊距：上 18mm / 下 15mm / 左右 15mm
內容寬度：511pt (180mm)
正文：9.5pt / leading 16pt
表格：8pt / leading 12pt
標題：H1=16pt / H2=13pt / H3=11pt
```

### 封面規則

```
主標題：最大 32pt，超過頁寬自動縮小
長標題：超過 15 個中文字 → 拆分主標題 + 副標題（14pt）
元素位置：品牌名 y=h-100、標題 y=h-260、副標 y=h-300
配色：深藍背景（#0F172A）+ 白字 + 金色分割線
```

### 表格規則

```
必須指定 col_widths（不指定會均分導致窄列）
欄寬總和 ≤ CONTENT_WIDTH (511pt)
表頭：深色背景 + 白字 + 粗體
數據行：斑馬紋交替色（白 + 淺灰）
數字：右對齊
文字：左對齊
漲跌幅：紅色(跌) / 綠色(漲) — 不用紅綠色盲友好時可加 ▲▼ 符號
```

### 圖表規則

```
圖表前後：必須有 4mm spacer
圖表標題：放在圖表上方，粗體
圖表尺寸：寬度 ≤ CONTENT_WIDTH，高度合理（不超過半頁）
配色：使用統一色板（深藍、淺藍、綠、橙、紅、灰）
標籤：必須有軸標籤和數據標籤
```

### 分頁規則

```
超過半頁的元素：不用 KeepTogether（會造成整頁空白）
章節之間：用 CondPageBreak(50) 而非硬 PageBreak
避免雙重空白：章節尾 + 新章節頭不要各一個 PageBreak
最後一頁：署名區 + 免責聲明，不留大片空白
```

### 自查（生成後自動執行）

```python
def pdf_self_check(pdf_path):
    """5 項品質檢查 — 全過才交付"""
    import fitz
    doc = fitz.open(pdf_path)
    issues = []
    for i, page in enumerate(doc):
        text = page.get_text()
        # 1. 空白頁
        if len(text.strip()) < 30:
            issues.append(f"第{i+1}頁疑似空白")
        # 2. 亂碼
        if "◆" in text or "■" in text:
            issues.append(f"第{i+1}頁有亂碼")
        # 3. 邊界（簡易）
        for b in page.get_text("dict").get("blocks", []):
            bbox = b.get("bbox", (0,0,0,0))
            if bbox[0] < 25 or bbox[2] > 570:
                issues.append(f"第{i+1}頁有元素超出邊界")
                break
    doc.close()
    if issues:
        for x in issues: print(f"  ❌ {x}")
        return False
    print("  ✅ PDF 自查通過")
    return True
```

## 第四部分：PPT 簡報規範

### 版面參數

```
尺寸：16:9（寬 13.333" × 高 7.5"）
安全區域：距邊界 0.5"（內容不超出）
字體：標題 ≥ 24pt、正文 ≥ 16pt、最小 13pt
每頁子彈點：最多 5 個
每個子彈點：最多 45 個中文字（超過截斷加「…」）
```

### McKinsey 風格結構

```
投影片 1：封面（標題 + 日期 + 作者）
投影片 2：Executive Summary（3-5 個要點）
投影片 3-N：每頁一個主題
  - Action Title（結論句，不是描述句）
  - 支撐數據（表格或圖表）
  - 底部要點（1 句話 takeaway）
最後一頁：結論 + 建議行動
```

### 設計規則

```
配色：≤ 3 種主色（深藍 + 白 + 灰，紅綠僅用於漲跌）
字體：≤ 2 種（一個標題字體 + 一個正文字體）
動畫：不用
漸層/陰影/3D：不用
圖片：高解析度，不拉伸變形
留白：元素之間 ≥ 0.3"
```

### Markdown 清理（文字入 PPT 前必執行）

```python
def clean_markdown(text):
    """PPT 不顯示 Markdown 語法"""
    if not text: return ""
    text = re.sub(r'\*\*(.*?)\*\*', r'\1', text)
    text = re.sub(r'\*(.*?)\*', r'\1', text)
    text = re.sub(r'\|(.*?)\|', r'\1', text)
    text = re.sub(r'#{1,6}\s*', '', text)
    text = re.sub(r'`(.*?)`', r'\1', text)
    return text.strip()
```

### 自查（生成後自動執行）

```python
def ppt_self_check(pptx_path):
    """PPT 品質檢查"""
    from pptx import Presentation
    from pptx.util import Emu
    prs = Presentation(pptx_path)
    issues = []
    for i, slide in enumerate(prs.slides):
        # 1. 空白頁
        has_text = any(s.has_text_frame and s.text_frame.text.strip()
                       for s in slide.shapes)
        if not has_text:
            issues.append(f"第{i+1}頁空白")
        # 2. 超出邊界
        for s in slide.shapes:
            if s.left + s.width > prs.slide_width + Emu(50000):
                issues.append(f"第{i+1}頁 '{s.name}' 超出右邊界")
            if s.top + s.height > prs.slide_height + Emu(50000):
                issues.append(f"第{i+1}頁 '{s.name}' 超出下邊界")
    if issues:
        for x in issues[:10]: print(f"  ❌ {x}")
        return False
    print("  ✅ PPT 自查通過")
    return True
```

## 第五部分：Email 規範（投行晨報標準）

### 設計哲學

投行晨報的核心：**5-10 分鐘讀完，讀者立刻知道該怎麼做。**

不是把 PDF 內容全塞進 Email — Email 是「電梯簡報」，PDF 才是「完整報告」。

### 文案寫作原則

| 原則 | 說明 | 範例 |
|:---|:---|:---|
| **結論先行** | 第一句話就是最重要的結論 | ✅「全球市場普跌，S&P -1.67%」❌「昨天的市場表現如下...」|
| **數據說話** | 每句話都要有數字支撐 | ✅「VIX 飆升至 31（+13%），恐慌情緒升溫」❌「市場波動加大」|
| **動作導向** | 告訴讀者該怎麼做 | ✅「建議等待 VIX 回落至 25 以下再進場」❌「市場有風險」|
| **簡短有力** | 主題行 ≤ 60 字元，正文 ≤ 150 字 | 摘要只寫 3-5 個要點 |
| **層次分明** | 掃一眼就知道重點在哪 | 粗體標重點、紅綠標漲跌、表格列數據 |

### 主題行格式

```
格式：[報告類型] | [最重要的訊息] | [日期]
範例：
  每日宏觀早報 | 全球普跌 S&P -1.67% VIX 31 | 2026-03-30
  股票篩選報告 | 美股 Top: NDAQ +7 MRK +7 | 2026-03-30
  賽馬分析 | 沙田 R5 孫子兵法 3.2x EV | 2026-03-30
```

### 防超連結處理（Gmail 會把 .HK .TW 識別為網址）

```python
def safe_ticker(ticker):
    """在後綴前插入零寬空格，防止 Gmail 自動變成超連結"""
    for suffix in [".HK", ".TW", ".T", ".SS", ".SZ"]:
        if ticker.endswith(suffix):
            return ticker.replace(suffix, "\u200B" + suffix)
    return ticker
```
